DISCUSSION

4. 議題

4.1 UIフレームワーク選定

React + Tailwind CSS の採用を決定。既存の社内ツールが React ベースであり、学習コストを抑えられることが最大の理由である。Vue.js も候補に挙がったが、社内ナレッジの不足から見送り。

採用技術スタック
FRAMEWORK
React
CSS
Tailwind CSS
DESIGN
Figma

李氏からは Tailwind CSS のデザインシステムとの親和性について説明があった。Figma で定義したカラーやスペーシングのトークンをそのまま CSS 変数として取り込めるため、デザイナーとエンジニア間で定義がずれる問題を解消できる。

4.2 検索機能の技術選定

全文検索バックエンドとして Elasticsearch を導入する方針。現行ポータルの SQL LIKE 検索ではレスポンスタイムが 3〜5 秒かかっており、改善要望が多い。

1
現行: SQL LIKE 検索 — レスポンスタイム 3〜5 秒
2
移行先: Amazon OpenSearch Service（Elasticsearch）
3
目標: レスポンスタイム 200ms 以内、追加コスト月額約 3 万円

4.3 デザイン方針の共有

李氏よりデザイン方針の説明。ユーザーリサーチの結果、現行ポータルに対して「情報が多すぎてどこを見ればよいかわからない」という声が全体の 68% を占めた。

◆
方針: ダッシュボード型トップページ
部署・役職に応じた情報のパーソナライズを行い、各ユーザーに必要な情報だけを表示する。ナビゲーションはグローバルヘッダーに集約し、階層は最大 2 段階。
パーソナライズ 階層: 最大2段
◆
方針: アクセシビリティ対応
カラーパレットはコーポレートカラーのブルー基調で、アクセシビリティ基準 WCAG AA を満たすコントラスト比を確保する。
WCAG AA 準拠 ブルー基調

# WBS・ガントチャート作成ガイド

## HTML 変換を前提とした frontmatter

md-to-html スキルで HTML に変換する場合は、YAML frontmatter に `type: wbs` を付与する。
HTML 変換時に進捗サマリーカード + CSS ガントチャート + WBS テーブルのレイアウトが適用される。

```yaml
---
type: wbs
title: プロジェクト WBS
project: プロジェクト名
date: YYYY-MM-DD
---
```

## WBS（Work Breakdown Structure）テンプレート

```markdown
---
type: wbs
title: プロジェクト WBS
project: プロジェクト名
date: YYYY-MM-DD
---

# [プロジェクト名] WBS

## 1. [フェーズ1名]

### 1.1 [作業項目]

- 成果物: [成果物名]
- 担当: [担当者]
- 期間: YYYY-MM-DD 〜 YYYY-MM-DD

### 1.2 [作業項目]

- 成果物: [成果物名]
- 担当: [担当者]
- 期間: YYYY-MM-DD 〜 YYYY-MM-DD

## 2. [フェーズ2名]

### 2.1 [作業項目]

...
```

## WBS をテーブル形式で書く場合

作業項目が多い場合はテーブルが見やすい。

```markdown
| No. | フェーズ | 作業項目 | 担当 | 開始 | 終了 | 状態 |
|---:|---|---|---|---|---|---|
| 1.1 | 要件定義 | ヒアリング | 田中 | 04/01 | 04/07 | 完了 |
| 1.2 | 要件定義 | 要件書作成 | 田中 | 04/08 | 04/14 | 作業中 |
| 2.1 | 設計 | 基本設計 | 佐藤 | 04/15 | 04/28 | 未着手 |
```

## WBS・ガントチャート固有のルール

- 見出し番号（1.1, 1.2, 2.1 ...）で階層を明示する
- 各作業項目に**担当者**と**期間**を必ず記載する
- マイルストーン（リリース日等）はテーブル内で明記する
- **スケジュールの視覚化は HTML 変換に任せる** — テーブルに開始日・終了日・状態を正確に書けば、HTML 変換時に CSS ガントチャートが自動生成される。Mermaid `gantt` は使わない
- コードブロックは原則不要

## 遅延・日程変更の表現ルール

WBS に遅延や日程変更がある場合、以下のパターンで統一的に表現する。

### Markdown 上での書き方

- **日付の変更**: 取消線で旧日付を示し、新日付を続ける — `~~2/21~~ 2/24`
- **遅延の注記**: 括弧で遅延幅を明示 — `（3日遅延）`
- **状態の記載**: 遅延タスクは状態列に `遅延中` や `3日遅延で完了` と記載
- **影響の説明**: テーブルの後に注記として遅延の影響を記述

### WBS テーブルでの表現

| 要素 | 表現方法 |
|---|---|
| 日付変更 | `~~旧日付~~ 新日付` で取消線表示 |
| 遅延で完了 | 状態列に `3日遅延で完了` |
| 遅延中 | 状態列に `遅延中` |
| 進捗バーの色 | 遅延タスクは赤（`--accent-red`）、順調は緑（`--accent-green`）、進行中は橙（`--accent-warm`） |
| フェーズ全体の遅延 | フェーズ行の状態を `遅延中` にし、進捗バーを赤にする |

### ステータスバッジの種類

| 状態 | Markdown 表記 | HTML クラス |
|---|---|---|
| 完了 | `完了` | `.badge.done` |
| 進行中 | `進行中` | `.badge.in-progress` |
| 未着手 | `未着手` | `.badge.not-started` |
| 遅延 | `遅延中` / `N日遅延で完了` | `.badge.delayed` |

### 例: 遅延を含む WBS テーブル行

```markdown
| 1.3 | 環境構築 | 鈴木 | 2/15 | ~~2/21~~ 2/24 | 100% | 3日遅延で完了 |
| 2.1 | 機能A開発 | 佐藤 | ~~2/25~~ 2/28 | ~~3/10~~ 3/13 | 60% | 遅延中 |
```

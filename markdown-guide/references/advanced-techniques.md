# 表現力を高めるテクニック

文書の内容や読者に合わせて、以下のテクニックを適切に使い分ける。

## 折りたたみ（Details / Summary）

長い補足情報、ログ出力、詳細な手順など **本文の流れを妨げる長い内容** は折りたたむ。

```html
<details>
<summary>クリックして詳細を表示</summary>

ここに折りたたむ内容を書く。

- 箇条書きもOK
- コードブロックもOK

</details>
```

使いどころ: トラブルシューティングの詳細手順、長いログ出力、オプション一覧

## タスクリスト（チェックボックス）

進捗管理や手順の達成状況を表す場面で使う。

```markdown
- [x] イベントコンセプトシート作成
- [x] クリエイティブへ共有
- [ ] ポジショニングステートメント起案
- [ ] ナビ台本作成
```

使いどころ: TODO リスト、作業チェックリスト、リリース前チェック

## テーブルの整列

数値は右寄せ、見出しは中央寄せなど、データの性質に合わせて整列を指定する。

```markdown
| 項目 | 説明 | 件数 |
|:---|:---:|---:|
| バグ | 不具合報告 | 1,234 |
| 機能要望 | 新機能リクエスト | 567 |
```

`:---`（左）、`:---:`（中央）、`---:`（右）で制御する。

## diff 表記（変更の Before / After）

変更前後の比較を明示する場合に使う。

````markdown
```diff
- 旧: Excelで手動管理
+ 新: 社内システムから自動取得
```
````

使いどころ: 設定変更の説明、移行手順、コードの修正箇所

## 脚注（Footnotes）

本文の流れを止めずに用語説明や参照を示す。

```markdown
設計仕様書はプロジェクト管理ツール[^1]を参照して作成する。

[^1]: プロジェクト管理ツールは社内の基幹システムです。
```

使いどころ: 専門用語の説明、出典の明示、補足情報

## 絵文字による視認性向上

見出しやリスト項目の先頭に絵文字を付けて、視認性を上げる。

```markdown
## 完了したタスク
## 作業中
## 未着手
## 重要なお知らせ
## 関連リンク
```

使いどころ: ステータス表示、セクションの区別、注目させたい箇所

## キーボードキーの表現

ショートカットキーや操作手順を示す場合に `<kbd>` タグを使う。

```markdown
プレビューを開くには <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>V</kbd> を押してください。
```

使いどころ: 操作手順書、ショートカット一覧

## 画像のサイズ制御

画像を挿入する際、サイズが大きすぎる場合は HTML タグで制御する。

```markdown
<!-- 通常（サイズ制御なし） -->
![図](image-url.png)

<!-- 幅を指定 -->
<img src="image-url.png" width="400">

<!-- パーセントで指定 -->
<img src="image-url.png" width="50%">
```

## 目次（アンカーリンク）

ドキュメントが長い場合、冒頭に目次を設ける。GitHub は見出しから自動的にアンカーが生成される。

```markdown
## 目次
- [概要](#概要)
- [作業フロー](#作業フロー)
- [注意事項](#注意事項)
```

使いどころ: セクションが5つ以上ある長いドキュメント

## テクニックの使い分け判断

| 状況 | 使うテクニック |
|:---|:---|
| 長い補足を隠したい | 折りたたみ |
| 進捗・達成状況を示す | タスクリスト |
| 数値データを並べる | テーブル（右寄せ） |
| 変更前後を比較する | diff 表記 |
| 専門用語を説明する | 脚注 |
| セクションを目立たせる | 絵文字 |
| 操作手順を示す | `<kbd>` タグ |
| 画像が大きすぎる | `<img>` タグ |
| ドキュメントが長い | 目次 |

# 比較資料作成ガイド

製品・サービス・技術などの複数の選択肢を並べて比較するドキュメント。
読み手が「違い」と「どれを選ぶべきか」を一目で把握できることが目的。

## HTML 変換を前提とした構造規約

この比較資料は md-to-html スキルで HTML に変換される場合がある。
HTML 変換時にカードグリッドレイアウトが適用されるよう、以下の規約に従う。

### frontmatter

必ず YAML frontmatter を付与する。

```yaml
---
type: comparison
title: 比較タイトル
subtitle: サブタイトル（任意）
---
```

### 見出しとカードの対応

HTML 変換時の構造マッピング:

| Markdown | HTML での役割 |
|---|---|
| `##` | カラム（比較カテゴリ） |
| `###` | カード（各プラン・選択肢） |
| `### 名前 -- 価格` | カード + 価格バッジ |
| `- 項目` | カード内のリスト |

### 構造テンプレート

```markdown
---
type: comparison
title: AI コーディングツール料金比較
subtitle: 2026年3月時点
---

## Claude

Claude Code を中心としたエコシステム。

### Free -- $0/月
- コード補完: 基本機能のみ
- チャット: 制限あり

### Pro -- $20/月
- コード補完: 無制限
- チャット: 無制限
- エージェント: 対応

### Max -- $100/月
- Pro の全機能
- 長時間エージェント実行
- 優先キュー

## Copilot

GitHub Copilot のプラン体系。

### Free -- $0/月
- コード補完: 2,000回/月
- チャット: 50回/月

### Pro -- $10/月
- コード補完: 無制限
- チャット: 無制限

## まとめ

> [!IMPORTANT]
> 比較結果の要約と推奨。

## 補足

注釈や前提条件。
```

> [!NOTE]
> `##` がカラム、`###` がカードになるため、比較対象ごとに `##` で分け、各プランを `###` で記述する。
> テーブルでの一括比較も併用可能だが、カードレイアウトの主体は `##` / `###` の構造。

## 比較パターン

### 並列比較（対等な選択肢を並べる）

すべての選択肢を同じ観点で比較する。料金比較やスペック比較に適している。

```markdown
## プラン比較

| 観点 | Free | Pro | Enterprise |
|---|---|---|---|
| 月額 | $0 | $10 | $39 |
| コード補完 | 2,000回/月 | 無制限 | 無制限 |
| チャット | 50回/月 | 無制限 | 無制限 |
| エージェント | - | 対応 | 対応 |
```

### 推奨比較（推奨を明示する）

選択肢に優劣がある場合、推奨・非推奨を明確に示す。技術選定や方式検討に適している。

```markdown
## 方式の比較

### 方式A — 推奨

> [!TIP]
> この方式を推奨する理由。

- メリット1
- メリット2

### 方式B — 非推奨

> [!WARNING]
> この方式が適さない理由。

- デメリット1
- デメリット2
```

### 世代・段階比較（時系列で並べる）

技術の進化や世代交代を示す場合。

```markdown
## 進化の流れ

| 世代 | 時期 | 代表的なツール | 特徴 |
|---|---|---|---|
| 第1世代 | 2021〜 | GitHub Copilot | コード補完 |
| 第2世代 | 2024〜 | Cursor, Windsurf | IDE統合チャット |
| 第3世代 | 2025〜 | Claude Code, Codex CLI | 自律型エージェント |
```

## 比較のための表現パターン

### 注釈の入れ方

比較表だけでは伝わらない補足情報は、表の直後に注釈として記載する。

```markdown
> [!NOTE]
> 上記の価格は2026年3月時点の情報。最新の料金は各公式サイトを参照。
```

### 観点の粒度が異なる場合

比較対象ごとにプラン体系が異なる場合は、無理に1つの表にまとめず、選択肢ごとにセクションを分ける。

## 図解の活用

比較資料では構成の違いを図で示すと効果的。図の種類の選び方は `diagram-patterns.md` を参照する。

| 場面 | 推奨する図 |
|---|---|
| アーキテクチャの違い | `flowchart` + `subgraph` を選択肢ごとに |
| 処理フローの違い | `flowchart LR` で並列配置 |
| 機能の有無比較 | テーブルの方が適切（図にしない） |

## 比較資料固有のルール

- **比較軸を揃える** — すべての選択肢を同じ観点で評価する
- **数値は右寄せ** — テーブルの数値列は `---:` で右寄せにする
- **推奨がある場合は明示する** — 読み手に判断を委ねるだけでなく、作成者の見解を示す
- **出典を明記する** — 料金や仕様は変わるため、情報源と調査時点を記載する
- **過度な装飾を避ける** — 比較表自体がわかりやすさの主役。図は補助的に使う

# メモ・調査ノート作成ガイド

## メモテンプレート

```markdown
# [テーマ・タイトル]

**日付**: YYYY-MM-DD
**目的**: [このメモの目的を 1 文で]

## 要点

- [結論・要点1]
- [結論・要点2]
- [結論・要点3]

## 詳細

[要点の補足や背景情報]

## 参考

- [リンクや出典]
```

## 調査ノートテンプレート

調査・技術検証の結果をまとめる場合:

```markdown
# [調査テーマ]

**日付**: YYYY-MM-DD
**調査者**: [名前]

## 背景・目的

[なぜこの調査を行ったか]

## 調査結果

### [項目1]

[調査内容と結果]

### [項目2]

[調査内容と結果]

## 比較（必要な場合）

| 項目 | 選択肢A | 選択肢B |
|---|---|---|
| [観点1] | [結果] | [結果] |
| [観点2] | [結果] | [結果] |

## 結論・推奨

[調査結果を踏まえた結論]

## 残課題

- [ ] [未解決の課題]
```

## メモ固有のルール

### 構成の原則

- **結論ファースト**: 要点・結論を冒頭に置き、詳細は後に書く
- メモは短さが価値 — 不要な前置きやフォーマルな挨拶は省く
- 箇条書きを積極的に使い、文章の壁を作らない

### 見出し構成

- `#` はテーマ（1 つだけ）
- `##` はセクション（要点、詳細、参考 等）
- `###` は必要な場合のみ

### 使い分け

| 場面 | 使うテンプレート |
|---|---|
| 会議の個人メモ | メモテンプレート |
| 技術調査の記録 | 調査ノートテンプレート |
| 比較検討 | 調査ノート + 比較テーブル |
| 思いつきの記録 | メモテンプレート（詳細セクション省略可） |

# 計画書作成ガイド

移行計画、導入計画、プロジェクト計画など、「何を・いつまでに・どの順序でやるか」を定義するドキュメント。
関係者間で作業の全体像・スコープ・スケジュールを合意するために使う。

## HTML 変換を前提とした frontmatter

md-to-html スキルで HTML に変換する場合は、YAML frontmatter に `type` を付与する。

- **縦スクロール型**（通常の計画書）: `type: report` を使用
- **プレゼン型**（説明・合意向け）: `type: presentation` を使用（`presentation-guide.md` を参照）

```yaml
---
type: report
title: 計画書タイトル
date: YYYY-MM-DD
---
```

## 計画書の種別ごとの補足

### 移行計画書

システム移行（OS、ミドルウェア、DB等）の計画では以下を特に重視する。

- **現行環境と移行先の対比表**: バージョン、設定値の差異を明示

```markdown
| 項目 | 現行環境 | 移行先 |
|---|---|---|
| OS | OL 6.10 | OL 9.4 |
| ミドルウェア | WAS 8.0.5 | WAS 8.5.5.28 |
| Java | 1.6 | 1.8 |
```

- **作業手順はコードブロックで**: コマンドやファイルパスを含む手順はコピペ可能にする
- **切り戻し手順は必須**: 移行計画には切り戻し計画を必ず含める

### プロジェクト計画書

プロジェクト全体の計画では WBS との組み合わせが有効。
WBS・ガントチャートの書き方は `wbs-gantt.md` を参照する。

## 図解の活用

計画書では工程の流れやシステム構成図が特に有効。図の種類の選び方は `diagram-patterns.md` を参照する。

| 場面 | 推奨する図 |
|---|---|
| 作業工程の全体像 | `flowchart TD` でフェーズを縦に並べる |
| システム構成の変更 | `flowchart` + `subgraph` で現行/移行先を対比 |
| スケジュール | テーブル形式で記述（HTML 変換時に CSS ガントチャートが自動生成） |
| 作業の依存関係 | `flowchart LR` で順序と分岐を表現 |

## 計画書固有のルール

- **メタ情報をヘッダーに置く** — 作成日・作成者・ステータス・対象を冒頭のテーブルで示す
- **スコープを明確にする** — 「対象範囲」と「対象外」を明示して認識のズレを防ぐ
- **作業項目にはチェックボックスを使う** — `- [ ]` で進捗を追跡可能にする
- **コマンドや設定値はコードブロックで** — コピペ可能にする
- **切り戻し計画を含める** — 特に移行計画では必須
- **承認欄を設ける** — 計画の合意・承認を記録する

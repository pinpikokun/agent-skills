# 検索ワークフロー

## Step 1: 絞り込み条件の収集

条件が少ないとタイムアウトする。**最低2つ以上**の条件を収集する:

| 条件 | 優先度 |
|------|--------|
| 期間（From〜To） | 最優先 |
| フォルダ | 最優先 |
| 検索対象（件名 / 本文 / 両方） | 最優先 |
| 件名キーワード / 送信者 | 推奨 |

条件が1つ以下なら、追加条件をユーザーに確認する:

> 現在の条件だけでは検索に非常に時間がかかるか、タイムアウトする可能性が高いです。以下の条件を追加で教えてください:
> - 期間（いつ頃のメールですか？）
> - フォルダ（受信トレイ？送信済み？）
> - 送信者または宛先
> - 件名に含まれるキーワード

条件が2つでも範囲が広い場合（期間1ヶ月以上、フォルダ指定なし等）は、期間を短くするか条件を追加するよう促す。

## Step 2: フォルダの特定

ユーザーがフォルダを指定しない場合、フォルダ一覧を取得して関連しそうな候補を推測・提示する。
複数選択可能であることを明記し、親フォルダ・受信トレイ直下も候補に含める。

## 人名検索

送信者名検索は**日本語名とローマ字名の両方**で検索する。Outlook の表示名はローマ字が多い。

## 複数フォルダの検索

検索対象が複数フォルダにまたがる場合は、サブエージェントを**複数並列起動**してフォルダごとに分割検索すると高速。

## 本文検索・フラグ検索が必要な場合

Python スキルの `search_body_hybrid()` / `search_flagged()` を使う方が高速で信頼性が高い。
繰り返す検索パターンがあれば、Python スキルの `scripts/` にスクリプト化を検討する。

PowerShell での実装が必要な場合のコード例は `code-examples.md` を参照。

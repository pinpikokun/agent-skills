# 検索ワークフロー

## Step 1: 絞り込み条件の収集

条件が少ないとタイムアウトする。**最低2つ以上**の条件を収集する:

| 条件 | 優先度 |
|------|--------|
| 期間（From〜To） | 最優先 |
| フォルダ | 最優先 |
| 検索対象（件名 / 本文 / 両方） | 最優先 |
| 件名キーワード / 送信者 | 推奨 |

条件が1つ以下なら、追加条件をユーザーに確認する:

> 現在の条件だけでは検索に非常に時間がかかるか、タイムアウトする可能性が高いです。以下の条件を追加で教えてください:
> - 期間（いつ頃のメールですか？）
> - フォルダ（受信トレイ？送信済み？）
> - 送信者または宛先
> - 件名に含まれるキーワード

条件が2つでも範囲が広い場合（期間1ヶ月以上、フォルダ指定なし等）は、期間を短くするか条件を追加するよう促す。

## Step 2: フォルダの特定

ユーザーがフォルダを指定しない場合:

1. フォルダ一覧を取得し、検索内容から関連しそうな候補を推測して提示する
2. **親フォルダも候補に含める**（振り分けが不完全で親に残っているケースがある）
3. **受信トレイ直下も候補から外さない**
4. 複数選択可能であることを明記する

サブフォルダを持つフォルダが選ばれたら、直下のみか再帰検索かを確認する。

## Step 3: キーワードの検索対象確認

キーワードが件名にあるか本文にあるかで検索方法が変わる。ユーザーに確認する:

> キーワード「○○」は件名に含まれますか？それとも本文から検索しますか？
> 1. 件名（高速：Restrict フィルタで検索）
> 2. 本文（日付で Restrict 後、Python 側でフィルタ）
> 3. 両方

## Step 4: 検索実行

サブエージェントに委任して検索を実行する。
検索対象が複数フォルダにまたがる場合は、サブエージェントを**複数並列起動**してフォルダごとに分割検索すると高速。

サブエージェントに検索を委任する際、**初回検索で必要な情報をすべて取得する**（2回目の検索は追加のトークンコストと時間がかかるため）:
- 件名、送信者名、送信者メールアドレス、受信日時、添付ファイル有無・件数、EntryID
- ペアリング（請求書+パスワード等）が必要な場合は、関連キーワードすべてを1回の検索に含める

## 人名検索

送信者名で検索する場合、**日本語名とローマ字名の両方で検索する**。
Outlook の送信者表示名はローマ字（例: `Taro Suzuki`）の場合が多く、日本語名だけではヒットしない。

- 日本語の姓 → ローマ字に変換して両方で検索
  - 例: 「佐々木」→ `LIKE '%佐々木%'` と `LIKE '%Sasaki%'` の2回検索
- ローマ字が不明 → メールアドレスに含まれるローマ字で検索（`苗字.名前@` のパターンが多い）
- **送信者名検索は全サブフォルダを再帰検索する**（人名フォルダに振り分けられている可能性が高い）
- 送信済みフォルダから同一人物への返信が見つかった場合、同一件名で受信フォルダ全体を再検索する

## 本文検索（ハイブリッド方式）

`client.search_body_hybrid()` メソッドを使用する。
内部で日付 Restrict → Python 本文フィルタを実行する。DASL の本文検索は非常に遅いため、この方式を使う。

## フラグ検索

`client.search_flagged()` メソッドを使用する。
内部で日付 Restrict → Python FlagStatus 判定を実行する。

- FlagStatus: 0=なし, 1=完了, 2=未完了フラグ
- フラグ付きメールは少数のため**全フォルダ再帰検索**が基本（`recursive=True` がデフォルト）
